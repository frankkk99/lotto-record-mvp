# Lotto Record MVP

เว็บแอพสำหรับบันทึกเลขสลาก 6 หลัก ตรวจคำนวณผลรางวัล และสรุปรายงานตามรายชื่อ

> ระบบนี้เป็นเครื่องมือบันทึกและตรวจคำนวณข้อมูลสลากส่วนตัวเท่านั้น ไม่ใช่ระบบรับแทง ไม่มีระบบฝากถอนเงิน ไม่มีระบบจ่ายรางวัล และไม่มีการชักชวนให้เล่นพนัน

## สิ่งที่ปรับปรุงล่าสุด

- ปรับหน้าใช้งานเป็น **Lotto Field Desk / โหมดหน้างาน** สำหรับคนจดรายการจริง
- เพิ่มแป้นตัวเลขใหญ่สำหรับกรอกเลขสลาก 6 หลักบนมือถือ
- เพิ่มปุ่มจำนวนใบแบบ `- / +` และปุ่มลัด `1, 2, 5, 10 ใบ`
- เพิ่มปุ่มลัดราคา/ใบ เช่น `80, 100, 120`
- เพิ่ม Preview ก่อนบันทึก: ชื่อ, เลข, จำนวนใบ, ยอดรวม
- เตือนรายการซ้ำ และรวมจำนวนใบให้อัตโนมัติเมื่อชื่อเดิม + เลขเดิม + ราคาเดิม
- ปรับรายการล่าสุดให้อยู่ข้างฟอร์ม เพื่อเช็กทันทีว่าบันทึกเข้าแล้ว
- ปรับหน้า `รายการ` ให้เหมาะกับมือถือ มีการ์ดรายการ กดปรับจำนวนใบได้
- ลบรายการแบบยืนยันก่อนเสมอ ลดโอกาสกดพลาด
- ยังรองรับวางข้อความจาก LINE, ตรวจผล, Report, Copy, Share LINE และ Export CSV

## จุดเด่นสำหรับใช้งานหน้างาน

1. **กดง่าย** — ปุ่มใหญ่ เหมาะกับมือถือ/แท็บเล็ต
2. **จดเร็ว** — กรอกชื่อครั้งเดียว แล้วบันทึกเลขต่อได้ทันที
3. **กันพลาด** — เลขต้องครบ 6 หลักก่อนกดบันทึก
4. **กันซ้ำ** — ถ้าชื่อเดิมเลขเดิม ระบบรวมจำนวนใบให้
5. **แก้ง่าย** — เพิ่ม/ลดจำนวนใบจากรายการได้ทันที
6. **สรุปง่าย** — ดูยอดรวม รายชื่อ จำนวนใบ และส่งสรุปเข้า LINE ได้

## Tech Stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- localStorage สำหรับ MVP
- Supabase-ready schema
- Vercel-ready deployment

## วิธีรันในเครื่อง

```bash
npm install
npm run dev
```

เปิดเว็บที่:

```bash
http://localhost:3000
```

## คำสั่งตรวจ build

```bash
npm run build
```

## วิธี Deploy ไป Vercel

1. เข้า Vercel
2. Import GitHub repo นี้
3. Framework เลือก Next.js
4. กด Deploy

## ต่อ Supabase จริง

1. สร้าง Supabase Project
2. เปิด SQL Editor
3. วางโค้ดจาก `supabase/schema.sql`
4. ตั้งค่า `.env.local` จาก `.env.example`
5. ย้ายข้อมูลจาก localStorage ไปใช้ Supabase client
6. เปิด Row Level Security ก่อนใช้จริง

## Roadmap รอบต่อไป

- Supabase Auth
- LINE Login / LIFF
- บันทึกข้อมูลงวดจริงลงฐานข้อมูล
- Role: admin / staff / viewer
- PDF export
- Audit log สำหรับการแก้ไข/ลบรายการ
- Backup/restore รายงวด
- ระบบค้นหาเร็วในหน้า `รายการ`
- โหมดพิมพ์ใบสรุป/ใบส่งลูกค้า

## หมายเหตุด้านความปลอดภัย

- เวอร์ชันนี้เก็บข้อมูลใน browser localStorage เพื่อทดสอบ UX ก่อน
- ห้ามใช้เก็บข้อมูลจริงระยะยาวจนกว่าจะต่อ Supabase และ Row Level Security
- ไม่ควรเพิ่มระบบฝากถอน จ่ายเงิน หรือฟีเจอร์รับแทงแบบสาธารณะ
- ควรตรวจผลรางวัลจากแหล่งทางการก่อนกดคำนวณทุกครั้ง
